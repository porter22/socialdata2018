1. Motivation

The dataset is taken from Copenhagen Open Data portal. It's a socioeconomic data about Copenhagen residents, which contains information about: income, age, education, origin distributions by districts. It is available in CSV and JSON formats.

Source: Socioeconomic data for Copenhagen, Denmark

I chose this dataset because I am relatively new to the city, and it was interesting to get an overview of it.

The goal for the user experience is to make users play with visualization and discover their own stories. Hopefully, users that are more familiar with the city might discover even more stories

2. Basic stats

The dataset was relatively clean. I've removed only several null strings and irrelevant columns.

The visualisation allowed me to discover interesting stories on-the-go, so I did not have to run any preliminary analysis. Once I've plotted the data onto Parallel Coordinates graph, the patterns (Story 1) and outliers (Stories 2 and 3) were evident visually.

3. Genre

I consider this visualisation to be a representative of Slide Show genre, according to Segal and Heer's taxonomy.

Visual Narrative tools used:

Highlighting - Feature distinction (to highlight districts both on the map and the graph)
Visual Structuring - Consistent Visual Platform (to tell different stories, but have some baseline visualisation).

Narrative Structure tools used:

Ordering - Linear (to adjust to slide show format)
Interactivity - Hover Highlighting, Filtering, Selection (to highlight districts on the map and the graph), Navigation Buttons (to switch between stories)
Messaging - Captions/Headlines, Annotations (to explain stories)

4. Visualizations

For the project I've chosen two interacting visualisations: Parallel Coordinates Plot and Choropleth map. Given the multidimensionality of the data, I've initially chosen the Parallel Coordinates Plot just to explore the dataset.
Once I've plotted the dataset, it became surprisingly easy to explore it using only visualisation. After that, the only missing thing was to add a geographical reference, so that the users could see the real world districts.
I believe that the Parallel Coordinates Plot was the right tool to quickly explore and visualize the data, while also preserving users ability to uncover own stories. The choropleth map was a rather standart choice, considering the granularity of the dataset.
